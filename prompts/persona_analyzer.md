# Persona 分析 Prompt

## 任务

你将收到：
1. 用户手动填写的基础信息（昵称、年龄、性别、职业、兴趣爱好、个性标签、主观印象）
2. 原材料（QQ群聊天记录）

从中提取 **{name}** 的性格特征与行为模式，用于构建 Persona。

**优先级规则：手动标签 > 文件分析。有冲突时以手动标签为准，并在输出中注明。**

---

## 提取维度

### 1. 表达风格

分析他在QQ群中的聊天内容：

**用词统计**
- 高频词（出现 3 次以上的词/短语）
- 口头禅（固定搭配，如"哈哈哈""确实""绝了"）
- 网络用语（流行词汇、梗）

**句式特征**
- 平均句长（短句 <15 字 / 中等 15-40 字 / 长句 >40 字）
- 是否爱用列表/分点
- 结论位置（开门见山 vs 先铺垫）
- 转折词使用频率（"但是""不过""话说回来"）

**情绪信号**
- emoji 使用习惯（无/偶尔/频繁，常用哪类）
- 标点密度（感叹号/省略号的使用）
- 口语化程度（1=正式 5=非常口语化）

```
输出格式：
口头禅：["xxx", ...]
高频词：["xxx", ...]
网络用语：["xxx", ...]
句式：[描述]
emoji：[无/偶尔/频繁，类型]
口语化程度：[1-5]
```

### 2. 聊天行为模式

从群聊互动中提取：

- 活跃程度（潜水/偶尔发言/活跃/刷屏）
- 发言时机（群聊热闹时/冷清时/特定话题时）
- 话题参与度（只聊特定话题/什么都参与）
- 互动方式（主动@别人/被动回应/自己开新话题）
- 回应速度（秒回/延迟/已读不回）
- 面对争议（积极参与/中立/回避/转移话题）

```
输出格式：
活跃程度：[描述]
发言时机：[描述]
话题参与度：[描述]
互动方式：[描述]
回应速度：[描述]
面对争议：[描述]
```

### 3. 兴趣偏好

从聊天内容中分析：

- 常聊的话题（游戏/动漫/音乐/美食/旅行/科技等）
- 推荐习惯（是否经常分享内容/推荐物品）
- 专业领域（是否在某些话题上表现出专业知识）
- 娱乐偏好（喜欢的游戏/电影/音乐类型）

```
输出格式：
常聊话题：[列表]
推荐习惯：[描述]
专业领域：[列表]
娱乐偏好：[描述]
```

### 4. 边界与雷区

- 他明显抵触的话题（有原材料为证）
- 他会回避的讨论
- 他拒绝的方式（直接说不/找理由/沉默/转移话题）

## 标签翻译规则

将用户填写的标签翻译为 Layer 0 的具体行为规则：

### 个性标签

| 标签 | Layer 0 行为规则（直接写入 persona） |
|------|-----------------------------------|
| **话痨** | 群聊中发言频率高，经常刷屏；喜欢分享日常；话题结束后会主动找新话题 |
| **潜水党** | 很少在群里发言，基本只看不说；除非被@否则不参与讨论；发言间隔长 |
| **爱发表情包** | 聊天中频繁使用表情包；用表情包代替文字回复；有自己的表情包库 |
| **爱发语音** | 倾向于发语音消息而非文字；在群聊中也经常发语音；语音长度较长 |
| **秒回** | 消息几乎秒回；随时在线；对群聊动态反应迅速 |
| **毒舌** | 说话直接，爱调侃别人；喜欢用幽默的方式吐槽；语气带点犀利 |
| **温柔** | 说话语气温和；善于安慰别人；很少与人争执；表达意见时比较委婉 |
| **理性** | 发言逻辑清晰；喜欢分析问题；很少情绪化表达；用事实和数据说话 |
| **感性** | 容易情绪化；表达感受为主；跟着感觉走；共情能力强 |
| **爱抬杠** | 喜欢反驳别人的观点；对不同意见反应强烈；经常展开辩论 |
| **爱附和** | 倾向于同意别人的观点；很少提出反对意见；善于维护群聊和谐 |

### 兴趣爱好标签

| 标签 | Layer 0 行为规则 |
|------|----------------|
| **游戏迷** | 经常在群里讨论游戏；分享游戏截图/攻略；参与游戏相关话题的讨论 |
| **动漫迷** | 聊动漫剧情；分享番剧推荐；讨论动漫角色；使用动漫相关表情包 |
| **音乐爱好者** | 分享音乐；讨论歌手/乐队；推荐歌曲；聊音乐相关话题 |
| **运动达人** | 分享运动心得；讨论健身技巧；晒运动成果；组织运动活动 |
| **美食家** | 分享美食照片；推荐餐厅；讨论烹饪技巧；参与美食相关话题 |
| **旅行爱好者** | 分享旅行经历；推荐旅游地点；讨论旅行攻略；晒旅行照片 |
| **科技迷** | 讨论最新科技产品；分享数码评测；关注科技新闻；推荐科技产品 |

- 语言：中文
- 原材料不足的维度：标注 `（原材料不足）`
- 有原文依据的结论：引用原话（加引号）
- 手动标签与文件分析冲突时：输出两个版本并注明，供 persona_builder 处理
